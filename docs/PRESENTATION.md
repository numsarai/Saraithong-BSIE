**วันที่:** เมษายน 2569

## 1. สรุปสำหรับผู้บริหาร (Executive Summary)

**BSIE** คือระบบวิเคราะห์ธุรกรรมทางการเงินอัจฉริยะ สร้างขึ้นเพื่อช่วยพนักงานสอบสวนตำรวจไทยในการ:

- **นำเข้า** รายการเดินบัญชีจากธนาคาร 8 แห่ง อัตโนมัติ (ไม่ต้องพิมพ์ข้อมูลเอง)
- **วิเคราะห์** เส้นทางเงิน ตรวจจับรูปแบบผิดปกติ หาความเชื่อมโยงข้ามบัญชี
- **แจ้งเตือน** เมื่อพบพฤติกรรมน่าสงสัย (ฟอกเงิน, แบ่งรายการ, บัญชีตัวกลาง)
- **สร้างรายงาน** พร้อมใช้งานส่งพนักงานอัยการ/ศาล

### ทำไมถึงต้องมีระบบนี้?

| ปัญหาปัจจุบัน | BSIE แก้ได้อย่างไร |
|--------------|-------------------|
| วิเคราะห์ statement ด้วยมือ ใช้เวลาหลายวัน | **ประมวลผลอัตโนมัติภายในนาที** |
| ดูได้ทีละบัญชี ไม่เห็นภาพรวม | **เชื่อมโยงข้ามบัญชี เห็นเครือข่าย** |
| ใช้ Excel ธรรมดา ไม่มีเครื่องมือวิเคราะห์ | **กราฟเครือข่าย, Timeline, แจ้งเตือนอัตโนมัติ** |
| ต้องซื้อ i2 Analyst's Notebook ราคาแพง | **มี mini i2 ในตัว ฟรี** |
| ไม่มีมาตรฐานรายงาน | **สร้าง PDF/Excel ตามมาตรฐาน พร้อมลงนาม** |

## 2. ความสามารถของระบบ (สำหรับผู้ฟังทั่วไป)

### 2.1 นำเข้าข้อมูลง่าย
- ลากไฟล์ statement เข้าระบบ — ตรวจจับธนาคารอัตโนมัติ
- รองรับ 8 ธนาคารไทย: SCB, KBANK, BBL, KTB, BAY, TTB, GSB, BAAC
- รองรับ Excel, PDF (แม้เป็นภาพสแกน), รูปภาพถ่าย
- ตรวจจับข้อมูลซ้ำอัตโนมัติ ไม่เก็บข้อมูลซ้ำซ้อน

### 2.2 วิเคราะห์อัจฉริยะ
- **กราฟเครือข่าย** — เห็นว่าเงินไหลจากใครไปใคร แบบ i2 Analyst's Notebook
- **Timeline** — เห็นรูปแบบธุรกรรมตามเวลา ตรวจจับช่วงเวลาผิดปกติ
- **ติดตามเส้นทางเงิน** — ตามเงินจากบัญชี A ไป B ไป C ได้สูงสุด 4 ทอด
- **ตรวจจับอัตโนมัติ** — 12 รูปแบบผิดปกติ (7 กฎกราฟ + 5 รูปแบบฟอกเงิน)
- **วิเคราะห์ทางสถิติ** — Z-score, IQR, กฎของ Benford, ค่าเฉลี่ยเคลื่อนที่

### 2.3 รายงานพร้อมใช้
- **Excel** — รายงานหลายชีท สีตามประเภท ฟอนต์ TH Sarabun New
- **PDF** — รายงานสอบสวนพร้อมหัว/ท้ายกระดาษ ช่องลงนาม
- **i2** — ส่งออกไฟล์เปิดใน i2 Analyst's Notebook ได้เลย
- **STR/CTR** — รูปแบบรายงาน ปปง.

### 2.4 ภาษาไทย
- UI ภาษาไทยทั้งระบบ สลับอังกฤษได้
- ฟอนต์ TH Sarabun New ในทุกรายงาน
- ตรวจจับชื่อไทย เบอร์โทร บัตรประชาชน PromptPay อัตโนมัติ

## 3. สถาปัตยกรรมทางเทคนิค (สำหรับนักพัฒนา)

### 3.1 Tech Stack

| ชั้น | เทคโนโลยี |
|------|----------|
| Backend | Python 3.12, FastAPI, 21 API routers, 34 services |
| Frontend | React 19, TypeScript, Vite, Tailwind CSS, Zustand |
| Visualization | Cytoscape.js (กราฟ), Recharts (Timeline) |
| Database | SQLAlchemy 2, SQLite (WAL mode), Alembic migrations |
| OCR | EasyOCR (Thai + English) |
| Authentication | JWT (python-jose), PBKDF2 password hashing |
| i18n | react-i18next (~500 translation keys) |
| CI/CD | GitHub Actions |

### 3.2 ขนาดโปรเจค

| รายการ | จำนวน |
|--------|-------|
| Python source files | 161 |
| TypeScript/React files | 41 |
| Total code | ~41,000 lines |
| API endpoints | 120+ |
| Database models | 20 tables |
| Automated tests | 244 (212 backend + 32 frontend) |
| i18n keys | ~500 (Thai + English) |

### 3.3 ความปลอดภัย

- JWT authentication + role-based access (admin/analyst/viewer)
- PBKDF2-SHA256 password hashing (100,000 iterations)
- Rate limiting (slowapi)
- Security headers (X-Frame-Options, CSP, etc.)
- File upload allowlist
- Max request body size (50 MB)
- Chain of custody / audit trail ทุก action
- SQLite file permissions 600

## 5. โครงสร้างพื้นฐานที่ต้องการ (Infrastructure)

### 5.1 Phase 1: Standalone (ปัจจุบัน)
```
[เครื่อง PC/Mac พนักงานสอบสวน]
  └── BSIE (localhost:8757)
      ├── FastAPI backend
      ├── React frontend
      └── SQLite database (local file)
```
**ไม่ต้องใช้ server ใดๆ — ทำงานบนเครื่องเดี่ยวได้เลย**

### 5.2 Phase 2: สถานีตำรวจ (Multi-user)
```
[Server สถานีตำรวจ / จังหวัด]
  ├── BSIE Backend (FastAPI + Uvicorn)
  ├── PostgreSQL Database
  ├── Nginx (reverse proxy + HTTPS)
  └── File Storage (evidence)
        │
        ├── [PC สอบสวน 1] ──── Browser ──── HTTPS
        ├── [PC สอบสวน 2] ──── Browser ──── HTTPS
        ├── [PC สอบสวน 3] ──── Browser ──── HTTPS
        └── [Tablet หัวหน้า] ── Browser ──── HTTPS
```

**Server ที่ต้องการ:**
- CPU: 4+ cores
- RAM: 16+ GB
- Storage: 500 GB SSD (สำหรับ evidence files)
- OS: Ubuntu 22.04 LTS
- HTTPS certificate (Let's Encrypt — ฟรี)

### 5.3 Phase 3-4: ระดับจังหวัด/ภาค
```
[Cloud / Data Center]
  ├── Load Balancer (Nginx)
  ├── BSIE Backend x 2-3 instances
  ├── PostgreSQL (primary + replica)
  ├── Redis (cache + session)
  ├── S3-compatible storage (evidence)
  ├── Backup server
  └── Monitoring (Prometheus + Grafana)
        │
        ├── [สถานี อ.เมือง] ──── VPN/HTTPS
        ├── [สถานี อ.เกาะสมุย] ── VPN/HTTPS
        ├── [สถานี อ.ไชยา] ──── VPN/HTTPS
        └── [บก.จว.สุราษฎร์ธานี] VPN/HTTPS
```
