### โมดูลของ SPNI Platform

| โมดูล | สถานะ | คำอธิบาย |
|-------|-------|---------|
| **BSIE — การเงิน** | ★ **v4.1 พร้อมใช้** | วิเคราะห์ statement ธนาคาร, ตรวจจับฟอกเงิน, กราฟเครือข่าย |
| CDR Analysis — โทรศัพท์ | 📋 วางแผน | วิเคราะห์ข้อมูลการโทร, หาเครือข่ายโทรศัพท์, Tower location |
| Social Media Intel | 📋 วางแผน | วิเคราะห์ social media, ตรวจจับบัญชีปลอม, เชื่อมโยงตัวตน |
| CCTV Analysis | 📋 วางแผน | วิเคราะห์ภาพ/วีดีโอ, face recognition, timeline |
| **Shared Layer** | ⚠️ บางส่วนมีแล้ว | Entity registry, Link analysis (มีใน BSIE), Case management |

### ทำไมเริ่มจาก BSIE (การเงิน)?

1. **คดีทางการเงินมีจำนวนมากที่สุด** — ฉ้อโกง, ฟอกเงิน, หลอกลวงออนไลน์
2. **ข้อมูลมีโครงสร้างชัดเจน** — statement ธนาคาร = ตาราง, ง่ายต่อการประมวลผล
3. **เห็นผลเร็ว** — ลดเวลาจาก 3-5 วัน เหลือ 30 นาที ต่อคดี
4. **พื้นฐานสำหรับโมดูลอื่น** — Entity registry, Link analysis ใช้ร่วมกันได้
5. **ไม่มีเครื่องมือฟรีที่ดีในตลาด** — i2 ราคาแพง, ไม่รองรับภาษาไทย

### การแชร์ข้ามโมดูล

เมื่อโมดูลอื่นถูกพัฒนา สามารถแชร์:
- **Entity Registry** — คนเดียวกันที่พบในทั้งบัญชีธนาคาร + CDR + social media
- **Link Analysis** — เชื่อมโยงข้ามโมดูล (เงิน → โทรศัพท์ → social → CCTV)
- **Case Management** — คดีเดียวรวมหลักฐานจากทุกโมดูล
- **Local LLM** — AI ช่วยวิเคราะห์ทุกโมดูล

## 1. สรุปสำหรับผู้บริหาร (Executive Summary)

**BSIE** คือโมดูลแรกของ SPNI Platform — ระบบวิเคราะห์ธุรกรรมทางการเงินอัจฉริยะ สร้างขึ้นเพื่อช่วยพนักงานสอบสวนตำรวจไทยในการ:

- **นำเข้า** รายการเดินบัญชีจากธนาคาร 8 แห่ง อัตโนมัติ (ไม่ต้องพิมพ์ข้อมูลเอง)
- **วิเคราะห์** เส้นทางเงิน ตรวจจับรูปแบบผิดปกติ หาความเชื่อมโยงข้ามบัญชี
- **แจ้งเตือน** เมื่อพบพฤติกรรมน่าสงสัย (ฟอกเงิน, แบ่งรายการ, บัญชีตัวกลาง)
- **สร้างรายงาน** พร้อมใช้งานส่งพนักงานอัยการ/ศาล

## 2. ความสามารถของระบบ (สำหรับผู้ฟังทั่วไป)

### 2.1 นำเข้าข้อมูลง่าย
- ลากไฟล์ statement เข้าระบบ — ตรวจจับธนาคารอัตโนมัติ
- รองรับ 8 ธนาคารไทย: SCB, KBANK, BBL, KTB, BAY, TTB, GSB, BAAC
- รองรับ Excel, PDF (แม้เป็นภาพสแกน), รูปภาพถ่าย
- ตรวจจับข้อมูลซ้ำอัตโนมัติ ไม่เก็บข้อมูลซ้ำซ้อน

### 2.2 วิเคราะห์อัจฉริยะ
- **กราฟเครือข่าย** — เห็นว่าเงินไหลจากใครไปใคร แบบ i2 Analyst's Notebook
- **Timeline** — เห็นรูปแบบธุรกรรมตามเวลา ตรวจจับช่วงเวลาผิดปกติ
- **ติดตามเส้นทางเงิน** — ตามเงินจากบัญชี A ไป B ไป C ได้สูงสุด 4 ทอด
- **ตรวจจับอัตโนมัติ** — 12 รูปแบบผิดปกติ (7 กฎกราฟ + 5 รูปแบบฟอกเงิน)
- **วิเคราะห์ทางสถิติ** — Z-score, IQR, กฎของ Benford, ค่าเฉลี่ยเคลื่อนที่

### 2.3 รายงานพร้อมใช้
- **Excel** — รายงานหลายชีท สีตามประเภท ฟอนต์ TH Sarabun New
- **PDF** — รายงานสอบสวนพร้อมหัว/ท้ายกระดาษ ช่องลงนาม
- **i2** — ส่งออกไฟล์เปิดใน i2 Analyst's Notebook ได้เลย
- **STR/CTR** — รูปแบบรายงาน ปปง.

### 2.4 ภาษาไทย
- UI ภาษาไทยทั้งระบบ สลับอังกฤษได้
- ฟอนต์ TH Sarabun New ในทุกรายงาน
- ตรวจจับชื่อไทย เบอร์โทร บัตรประชาชน PromptPay อัตโนมัติ

### 5.3 ทำไมต้อง Local LLM (ไม่ใช้ Cloud AI)

> **ข้อมูลคดีเป็นชั้นความลับ** — ไม่สามารถส่งข้อมูลธุรกรรม ชื่อผู้ต้องสงสัย เลขบัญชี หรือเส้นทางการเงินไปยัง Cloud AI (ChatGPT, Claude API, Gemini) ได้ เพราะ:
>
> 1. **ความลับทางราชการ** — ข้อมูลคดีอาจเป็นความลับระดับ "ลับ" หรือ "ลับมาก"
> 2. **พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล** — ข้อมูลบัญชีเป็นข้อมูลส่วนบุคคลที่มีความอ่อนไหว
> 3. **Training Data Risk** — Cloud LLM อาจนำข้อมูลไปเรียนรู้ (training) โดยไม่ได้รับอนุญาต
> 4. **Chain of Custody** — หลักฐานต้องไม่ผ่านระบบภายนอกที่ไม่ได้รับการรับรอง
> 5. **ระเบียบ สตช.** — ข้อมูลสืบสวนต้องอยู่ในโครงข่ายภายในหน่วยงาน

### 5.4 Local LLM ทำอะไรได้บ้าง

| ความสามารถ | ตัวอย่าง |
|-----------|---------|
| **สรุปคดีอัตโนมัติ** | "พบเงินเข้าจาก 15 บัญชีรวม 3.2 ล้านบาท โอนต่อไป 3 บัญชีภายใน 24 ชม." |
| **ตอบคำถามเกี่ยวกับคดี** | "บัญชีนี้มีธุรกรรมกับใครบ้างเกิน 1 แสนบาท?" |
| **จำแนกประเภทธุรกรรม** | ช่วย classify ธุรกรรมที่ heuristic ไม่มั่นใจ |
| **แปลรายละเอียด** | แปลรายการภาษาอังกฤษเป็นไทยหรือกลับกัน |
| **ตรวจจับรูปแบบ** | "ช่วยดูว่ามีรูปแบบ structuring หรือไม่?" |
| **ร่าง narrative** | ร่างเนื้อหาสำหรับรายงาน STR/CTR |
