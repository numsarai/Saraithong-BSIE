# BSIE Graph System Architecture (v4.1)

> Consolidated from: graph-export.md, graph-foundation-phase1.md, graph-suspicious-rules-phase2.md, i2-export-workflows.md

## Overview

The BSIE graph system transforms normalized transactions into an investigation-grade network model. It provides:

1. **Graph Foundation** — deterministic node/edge model from transactions
2. **Graph Analysis** — SNA metrics, connected components, suspicious pattern detection
3. **Alert System** — 7 graph rules + 5 threat hunting patterns
4. **Visualization** — LinkChart (Cytoscape.js), AccountFlowGraph, TimeWheel
5. **Export** — CSV, JSON, i2 ANX, i2 XIMP, graph_analysis.xlsx

## Architecture

```
Transactions (normalized)
    ↓
core/graph_export.py → nodes_df + edges_df + aggregated_edges_df
    ↓
core/graph_analysis.py → SNA metrics, connected components, findings
    ↓
core/graph_rules.py → 7 suspicious pattern rules
    ↓
services/alert_service.py → persistent Alert records
services/threat_hunting_service.py → 5 FATF-inspired patterns
services/sna_service.py → degree/betweenness/closeness centrality
    ↓
routers/graph.py → API endpoints for nodes, edges, findings, neighborhood
routers/analytics.py → SNA, anomaly, threat hunt endpoints
    ↓
LinkChart.tsx → multi-hop interactive visualization (Cytoscape.js)
AccountFlowGraph.tsx → radial flow visualization
```

## Graph Model

### Nodes
- **Account nodes** — subject + counterparty accounts
- **Entity nodes** — people, organizations from NLP extraction

### Edges
- **Flow edges** — individual transaction flows (SENT_TO, RECEIVED_FROM)
- **Aggregated edges** — consolidated account-to-account flows with totals
- **Derived edges** — account-level relationships

### Columns
- nodes: 25 columns (id, type, label, account_number, entity_name, bank, etc.)
- edges: 38 columns (id, from/to, amount, date, confidence, lineage, etc.)

## Detection Rules (7 graph rules)

| Rule | Detects | Severity |
|------|---------|----------|
| repeated_transfers | Same pair ≥3 transfers | medium |
| fan_in_accounts | Multiple sources ≥3 | high |
| fan_out_accounts | Multiple targets ≥3 | high |
| circular_paths | Bidirectional flow | high |
| pass_through_behavior | Intermediary account | high |
| high_degree_hubs | ≥6 connections | medium |
| repeated_counterparties | Same CP multiple days | medium |

## Threat Hunting (5 FATF patterns)

| Pattern | Detects |
|---------|---------|
| Smurfing | Deposits split below 200K threshold |
| Layering | High counterparty turnover per day |
| Rapid Movement | In→out within 4 hours |
| Dormant Activation | 60+ day gap then sudden activity |
| Round-Tripping | Funds sent and returned from same source |

## i2 Export

### ANX Chart (.anx)
- XML format for i2 Analyst's Notebook
- Contains nodes + edges from graph model
- Generated by `core/export_anx.py`

### Import Package (.csv + .ximp)
- CSV data with 29 columns (From/To Account, Amount, Date, etc.)
- XIMP XML spec for i2 import wizard
- Generated by `core/export_i2_import.py`
